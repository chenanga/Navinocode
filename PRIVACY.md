# Navinocode 隐私政策

生效日期：2026 年 7 月 14 日

Navinocode 是一个替换 Microsoft Edge 新标签页的效率导航扩展。本政策说明扩展在运行过程中保存和传输的数据。

## 本地保存的数据

以下数据保存在用户自己的浏览器扩展存储空间中：

- 快捷应用名称、网址、图标和排序
- 搜索历史与本地使用热力图事件
- 待办事项和番茄钟设置
- 背景图片、主题和界面设置
- 小组件位置和固定状态
- 用户主动填写的 Supabase URL、anon key 和同步 ID

这些数据默认不会发送给 Navinocode 项目维护者。用户可以通过设置中的配置导出功能备份部分数据，也可以通过清除扩展数据或卸载扩展删除本地数据。配置导出文件不包含 Supabase URL 或 anon key。

## 第三方服务和数据传输

扩展会在提供对应功能时访问以下第三方服务：

- Bing：用户在搜索框输入内容时，将搜索文本发送到 Bing，以获取搜索建议。
- 用户选择的搜索引擎：用户确认搜索后，搜索内容会通过新标签页发送到 Google、Bing、百度或 DuckDuckGo。
- 一言：请求随机短句作为搜索框占位文本，请求不包含用户输入内容。
- GitHub：读取 Navinocode 公开仓库的 Star 数量，请求不包含扩展内保存的用户内容。
- 背景和网站图标服务：加载随机背景、用户指定的图片或网站图标时，对应服务会收到常规网络请求；自动图标服务可能收到用户添加的应用网址。
- Supabase：仅当用户主动配置并启用云同步后，扩展才会把应用列表、待办、组件设置、搜索引擎和外观配置发送到用户指定的 Supabase 项目。搜索历史和使用热力图事件不包含在云同步数据中。

这些服务可能按照各自的隐私政策处理 IP 地址、User-Agent、请求时间等常规网络信息。扩展仅通过 HTTPS 访问远程服务。

## 不进行的行为

Navinocode：

- 不包含广告或第三方分析 SDK。
- 不出售用户数据。
- 不下载或执行远程代码。
- 不读取浏览历史、书签、Cookie 或其他网页内容。
- 不主动收集密码、支付信息、健康信息等敏感数据。

用户不应在待办、应用配置、同步 ID 或其他可同步字段中保存敏感信息。

## 权限说明

扩展申请 `unlimitedStorage`，用于在本地保存用户上传的背景图片和其他个性化配置。主机权限仅用于 Bing 建议、一言、GitHub 公开数据和用户主动配置的 Supabase 云同步。

## 用户控制

用户可以：

- 关闭自动云同步或删除 Supabase 配置。
- 导出自己的本地配置。
- 通过 Microsoft Edge 的扩展数据管理功能清除搜索历史和其他扩展站点数据。
- 在 Microsoft Edge 中卸载扩展，从而删除扩展的本地数据。

已经发送到用户自有 Supabase 项目的数据，需要由用户在对应 Supabase 项目中管理或删除。

## 政策更新

扩展功能或数据处理方式发生变化时，本政策会同步更新，并修改生效日期。重大变化会在版本说明或扩展界面中说明。
